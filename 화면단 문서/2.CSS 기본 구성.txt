**CSS 우선순위 규칙

Cascading은 작은 계단 모양의 폭포를 말한다
연속적인 물의 흐름에서 높낮이 차이를 우선순위에 비유

CSS는 하나의 태그에 여러가지 방법으로
동일한 속성의 스타일을 적용하는 경우 
우선순위에 따라서 적용될 스타일이 결정된다 

속성을 적용하다 보면 하나의 태그에 본의 아니게
같은 속성이 겹쳐 적용될 때가 있는데
우선순위에 의해 결정이 된다

*우선순위 결정 요소
1.중요도
2.명시도
3.코드의 순서

저작자 !important > 저작자 CSS > 사용자 CSS > 브라우저 CSS

아무것도 적용하지 않으면 브라우저 CSS가 적용된다

*선택자에 따른 명시도
inline style > id > class > 태그
태그 요소에 스타일 속성을 줄 때 선택자를 사용한다.
명시도는 더 구체적이라는 뜻이고 구체적일 수록 명시도가 높다
ex: id선택자는 class 선택자보다 명시도가 높고 
class 선택자는 태그 선택자 보다 명시도가 높다
같은 태그 선택자끼리는 뒤에 추가적인 선택자가 더 붙는 것이 더 자세하므로
명시도가 높다


*같은 선택자 끼리라면 깊이가 깊을수록(구체적) 명시도가 높다


*코드 순서
선택자의 종류와 깊이가 같을 때 우선순위의 결정 방식은 코드의 순서이다

위에서 아래로 코드가 수행되므로 위의 내용들 모두가 동일한 우선순위라면
제일 아래에 있는 코드가 적용되는 코드이다.



*display
태그의 성격을 바꿔주는 속성으로 display가 어떻게 설정되어 있는지에 따라 
그 후 사용할 수 있는 속성이 달라지므로 주의를 기울여야 한다

html 태그들은 block 또는 inline의 기본값들을 가지고 있는데,
그 기본값을 변경해 줄 수 있다

block			inline을 block로 변경함 -> 줄바꿈 일어남
inline			block을 inline으로 변경함 -> 줄바꿈이 일어나지 않음
inline-block	inline태그의 성격을 inline-block로 변경함 -> 줄바꿈은 일어나지 않으나
				block 태그만 사용할 수 있는 text-aline 등의 속성들을 사용할 수 있게 됨
none			태그를 보이지 않게 숨기고 흔적도 숨김

*background는 대충 color과 image만 기억하자

*padding
태그의 안쪽 여백
즉, 내용과 테두리 사이의 여백을 의미한다

*margin
태그의 바깥 여백
즉, 테두리와 다음 박스 사이의 여백을 의미한다
margin은 width에 포함되지 않으며, 블록 태그에 width값을 명시하지 않고
margin 좌우를 주면 가로 길이가 줄어들게 된다


*레이아웃1: float
block 태그는 원래 세로로만 나열되지만 float을 이용하면 가로로 배치할 수 있다

float을 지정하는 방법과 함께 float을 해제하는 방법도 알아야 한다

float은 태그를 좌측이나 우측에 붙이고 아래 내용이 그 주변을 흐르게 하는 속성이다

left속성값: 태그를 왼쪽에 배치하고 나머지 컨텐츠는 그 주변을 흐르도록 함
right속성값: 태그를 오른쪽에 배치하고 나머지 컨텐츠는 그 주변을 흐르도록 함
주의사항: float은 가로 



*clear
float된 박스들의 바로 아래 박스에게 주변을 흐르지 않고 원래대로
배치되도록 하는 속성이다.
float을 해제시켜주는 중요한 역할을 하는데
이 태그는 블록이어야 한다

쓸수 있는 값		설명
left			float된 박스 중 좌측이 짧을 때 좌측의 빈 공간부터 채워 내려옴
right			float된 박스 중 우측이 짧을 때 우측의 빈 공간부터 채워 내려옴
both			float된 박스 중 어느 쪽도 채우지 않고 다시 한단으로 배치해줌(가장 중요)
